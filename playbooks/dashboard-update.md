# Command Centre Dashboard Playbook
_Updated after each run._

## DASHBOARD RULES (NON-NEGOTIABLE)
1. NO GREEN TICKS on the dashboard. Ever. Unless Adam has personally opened and actioned the item.
2. Completed/actioned items get REMOVED from the dashboard entirely. Log them in memory/task-log.md.
3. When Adam responds to something, REMOVE it from the dashboard.
4. Dashboard must fit ONE SCREEN. If it doesn't, trim.
5. Past dates in UPCOMING get removed automatically.
6. Fleet status shows plain text status - no ticks, no emojis for completion.
7. Items only move through: NEEDS ADAM -> IN PROGRESS -> REMOVED (logged)
8. Reports from agents stay in OVERDUE/ALERTS or IN PROGRESS until Adam opens them.

## Proven Process
1. Read each agent workspace output/ and dev/ for new deliverables
2. Check for overdue items (past deadline)
3. Update output/dashboard.md following rules above
4. Track: PENDING -> Adam opens -> REMOVED
5. Monitor token spend thresholds

## Dashboard Sections
- NEEDS ADAM NOW - requires his decision
- OVERDUE / ALERTS - past deadline or failing
- IN PROGRESS - active work happening
- FLEET STATUS - agent health (plain text, no ticks)
- UPCOMING - next 7 days only
